    Sobre Aplicaciones Móviles (Progresivas - Híbridas - Nativas)

    Cuando se habla de desarrollar una aplicación móvil, se infiere que dicha aplicación va destinada a dispositivos diferentes a PC y portátiles convencionales, que por lo general va enfocada a dispositivos tipo celular.
    Si hablamos de aplicaciones progresivas son estas que desarrollan para web pero que se adecuan a la pantalla bajo la cual se visualizan sin perder la distribución de los elementos propios del front pero se ejecutan desde un browser del dispositivo siendo accesible a través de este

    Cuando se habla de aplicaciones nativas nos referimos a todas aquellas desarrolladas exclusivas para el sistema (hardware - software) destino, es decir, que se programó teniendo en cuenta las características propias de ese tipo de dispositivos por ejemplo nativas a sistemas operativos móviles como IOs y android.

    Finalmente cuando se habla de aplicaciones híbridas nos referimos a ese conjunto compartido de características, regularmente diseñada en entornos de desarrollo web con gestores de instalación multiplataforma y son más fáciles de gestionar mantenimiento.

    ¿qué es un framework de desarrollo móvil?

    Los frameworks de desarrollo móvil son aquellos softwares enriquecidos con una serie de bibliotecas optimizadas para la gestion y diseño de aplicaciones móviles. Estos frameworks permiten crear aplicaciones que funcionan en varias plataformas (Android e iOS).

    Frameworks de Desarrollo Móvil (Híbridas)

    Algunos de los frameworks más conocidos para el desarrollo de aplicaciones móviles con enfoques híbrido son:

    React Native: Utiliza JavaScript y React para crear aplicaciones nativas multiplataforma.

    Flutter: Utiliza el lenguaje Dart y ofrece una experiencia de desarrollo rápida y fluida para entornos multiplataforma IOS y Android.

    Librerias usadas con Javascript para el consumo APIs

    Axios es una de las libreiras mas usadas de mayor amplitud utilizada para la realización de peticiones HTTP, es decir, consumo servicios web o APIs.

    jQuery es tambien famosa por simplificar la manipulación del DOM (Document Object Model), lo que facilita la gestiónar eventos y la ejecución de peticiones AJAX.

    Las dos librerias ejecutan los métodos ya conocidos GET, POST, PUT, DELETE, etc., haciendolo de forma sencilla, son livianas e intuitivas, aunque su uso ha ido decreciendo debido al auge de frameworks modernos como React y Vue.js.

    aplicaciones de servicios web vs aplicaciones tradicionales

    Aunque pueden ser funcionales las dos metodologias de desarrollo, cuando se habla de desarrollo por servicios web se infiere una mayor intregalidad con servicios externos que ofrecen los mismo que se quiere ofrecer.
    Es preciso entender que el desarrollo de una aplicacion no siempre implica crear la rueda nuevamente, sino hacer uso de otras ruedas ya existes para echar a andar nuestro desarrollo.
    No tengo porque crear un boton nuevamente si alguien ya lo hizo como deseo, lo importante es a donde me dirige ese botón.
